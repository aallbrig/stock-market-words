# Ticker Lookup Enhancements

**Status:** Accepted
**Author:** Andrew Allbright
**Created:** 2026-04-24
**Supersedes:** —
**Superseded by:** —

## Context

The "Look Up a Ticker" form on the home page (rendered by the
`ticker-portfolio-extraction-tool` shortcode) is a bare-bones text input →
Go button → redirect. Three friction points hurt the user experience:

1. **No discoverability.** Users must know the exact ticker symbol. There is
   no way to search by company name, which is how most people think about
   stocks ("What is Apple's ticker?").
2. **Silent 404s.** An unknown or mistyped symbol silently navigates to a
   404 page with no explanation.
3. **Re-typing friction.** Users who look up several tickers in a session
   must re-type each one from scratch.

All three fixes share a single client-side data load (a new lightweight
`ticker-lookup.json`) and a single new script (`ticker-lookup.js`).

## Goal

The "Look Up a Ticker" input on the home page offers real-time autocomplete
by symbol and company name, prevents navigation to invalid tickers with an
inline error message, and surfaces the user's last 5 lookups as
quick-access chips below the input.

## Non-goals

- Autocomplete on any page other than the home page shortcode.
- Server-side or API-based search.
- Fuzzy / approximate matching (typing "APPL" does not suggest "AAPL").
  Deferred to a follow-up spec.
- Persisting lookups across devices or browsers.
- Internationalising the error message and "Recent:" label (English only
  for v1; symbols and company names are English regardless of language).
- Modifying the Extract Portfolios form.
- Sorting autocomplete by popularity or score.

## User stories

- As a first-time visitor, I want to type "Apple" in the lookup input and
  see "AAPL — Apple Inc." appear in a dropdown so I can navigate to the
  right ticker page without knowing the symbol beforehand.
- As a visitor who typed a symbol that doesn't exist, I want to see an
  inline "was not found" warning instead of landing on a 404 page.
- As a returning visitor, I want to see the last 5 tickers I looked up
  as chips below the input so I can revisit them in one click.

## Design

### 1. New data file: `ticker-lookup.json`

A minimal JSON file served to the browser, generated by the Python
pipeline:

```json
{
  "_meta": { "auto_generated": true, ... },
  "total": 5789,
  "tickers": [
    { "s": "AAPL", "n": "Apple Inc." },
    { "s": "MSFT", "n": "Microsoft Corporation" }
  ]
}
```

Short keys (`s`, `n`) keep the payload small. With ~5 800 entries and
compact serialisation the file is approximately 200–250 KB uncompressed
and ~55–70 KB gzipped (browsers cache it after first load).

**Generator:** Add `generate_ticker_lookup_json()` to
`python3/src/stock_ticker/hugo_generators.py`. The function queries all
tickers that have at least one `daily_metrics` row with a non-null price,
writes compact JSON to `HUGO_STATIC_DATA_DIR / "ticker-lookup.json"`, and
is called as the final step of `generate_all_hugo_content()`.

**Output path:** `hugo/site/static/data/ticker-lookup.json`

### 2. New script: `ticker-lookup.js`

A self-contained IIFE at `hugo/site/static/js/ticker-lookup.js`.

#### A. Lazy data loading

```js
function loadData() {
  // fetches SITE_BASE_URL + 'data/ticker-lookup.json' once,
  // caches result in module-level variable.
}
```

Data is fetched on the first `focus` event on `#ticker-lookup-input`.
Subsequent calls return the cached result immediately.

#### B. Autocomplete dropdown

- Listens to `input` events on `#ticker-lookup-input`.
- After data loads, filters `tickers` where `s.startsWith(QUERY_UPPER)`
  **or** `n.toLowerCase().includes(query_lower)`, capped at 8 results.
- Renders a Bootstrap `dropdown-menu` anchored to a `position: relative`
  wrapper around the input element.
- Each item shows `<strong>SYMBOL</strong> <span>Company Name</span>`.
- Keyboard navigation: `ArrowDown` / `ArrowUp` cycle items; `Enter`
  selects; `Escape` dismisses. Clicking an item fills the input and
  submits.
- Dropdown closes on outside click.

#### C. Validation before navigation

Replaces the inline `tickerLookup()` in the shortcode template:

```js
function submitLookup(sym) {
  sym = sym.trim().toUpperCase();
  if (!sym) return;
  if (lookupData && !lookupData.some(t => t.s === sym)) {
    showError('"' + sym + '" was not found. ...');
    return;
  }
  saveRecent(sym);
  window.location.href = SITE_BASE_URL + 'tickers/' + sym.toLowerCase() + '/';
}
```

If `lookupData` has not yet loaded, the check is skipped and the user
navigates immediately (same behaviour as today). Errors display in a
Bootstrap `alert-warning` `<div id="ticker-lookup-error">` inserted below
the form.

#### D. Recent lookups

- `saveRecent(sym)` — prepends `sym` to a `localStorage` array under
  key `smw-recent-tickers`, capped at 5 unique entries.
- `renderRecent()` — reads `localStorage` and renders Bootstrap badge
  links in `<div id="ticker-recent-lookups">`. Container is hidden when
  the array is empty. Called on `DOMContentLoaded` and after each save.

### 3. Template changes

**File:** `hugo/site/layouts/shortcodes/ticker-portfolio-extraction-tool.html`

1. Load the new script (same pattern as `portfolio-extractor.js`):
   ```go-html
   <script src="{{ .Page.Site.BaseURL }}js/ticker-lookup.js" defer></script>
   ```
2. Wrap `#ticker-lookup-input` in a `position: relative` div so the
   dropdown can be absolutely positioned below it.
3. Remove `maxlength="10"` from the input — users may type long company
   names.
4. Add `<div id="ticker-lookup-error" style="display:none;"></div>`
   below the form.
5. Add `<div id="ticker-recent-lookups" style="display:none;"></div>`
   below the error div.
6. Remove the inline `onsubmit="return tickerLookup(event)"` attribute
   from the form — `ticker-lookup.js` attaches its own `submit` listener.
7. Remove the inline `<script>` block containing the old `tickerLookup()`
   function.

## Verification

### Manual

1. `hugo server` → home page → type "APP" → dropdown shows "AAPL —
   Apple Inc." and other `APP*` matches.
2. Type "apple" (lowercase) → same (or similar) matches appear — name
   search works.
3. Type "ZZZZZ" → click **Go** → inline "was not found" warning appears;
   URL does not change.
4. Look up "AAPL" successfully → the "AAPL" chip appears below the input
   on next page load / after navigation.
5. Look up 6 different tickers → only the 5 most recent chips appear.
6. `/zh-cn/` home page → all three features work (script is
   language-agnostic; `SITE_BASE_URL` is set correctly by the shortcode).

### Automated

New file: `tests/playwright/ticker-lookup.spec.js`

- **Test 1** — Autocomplete by symbol: type "AAPL" → dropdown visible,
  contains "AAPL".
- **Test 2** — Autocomplete by name: type "apple" → dropdown visible,
  at least one item contains "Apple".
- **Test 3** — Invalid ticker: type "ZZZZZZ" → submit → error `div`
  visible; page URL unchanged.
- **Test 4** — Recent chips: pre-populate `localStorage` with
  `["MSFT","AAPL"]` via `page.evaluate`, reload, assert both chips

Run `npm run test:e2e` — all existing Playwright tests pass.

## Open questions

1. **Autocomplete on mobile.** `autocapitalize="characters"` is already
   set and aids iOS symbol entry. The dropdown may need a responsive
   max-width. **Default:** proceed and fix display issues if reported.

2. **Data file staleness.** `ticker-lookup.json` is regenerated only when
   the pipeline runs (once per trading day). Newly listed tickers appear
   with a one-day delay. **Default:** acceptable — same cadence as all
   other data files.

3. **What to do if localStorage is unavailable** (private browsing,
   storage quota exceeded). **Default:** catch the error silently; recent
   lookups simply don't persist.

## Alternatives considered

- **`<datalist>` instead of custom dropdown** — native, zero JS, but
  offers no keyboard navigation, no two-column symbol+name rows, and no
  company-name search in all browsers. Rejected for UX quality.
- **Reuse `filtered_tickers.json`** instead of a new file — avoids a
  Python change but the file is 2.5 MB and contains dozens of unused
  fields. A dedicated `ticker-lookup.json` (~200 KB compact) is cleaner.
- **Fuse.js for fuzzy search** — better typo tolerance but adds a ~10 KB
  dependency. Deferred to a follow-up spec.
