## Production parameters

The Hugo site reads two environment variables for production-only features.
Both default to empty (so local dev works without them):

| Env var | Hugo param | Purpose |
|---|---|---|
| `HUGO_PARAMS_GOOGLEANALYTICSID` | `googleAnalyticsId` | GA4 measurement ID. Empty in local dev. |
| `HUGO_PARAMS_GOOGLEADSENSEID` | `googleAdSenseId` | AdSense client ID. Empty in local dev. |

These are wired in `hugo/site/hugo.toml` `[params]` and consumed in
`hugo/site/layouts/partials/google-analytics.html` and
`google-adsense.html`. The conditional partials no-op when the param is empty,
so local builds never load tracking scripts.

## Trading-day cadence

US markets close at 16:00 ET. The intended cadence is:

1. After close, the data pipeline runs (`ticker-cli run-all`).
2. The Hugo site is rebuilt with the fresh data.
3. The new build is deployed.

## Health checks

After a deploy, sanity-check:

- `https://stockmarketwords.com/` loads and the Extract Tickers tool returns results
- `https://stockmarketwords.com/tickers/crm/` resolves with today's price
- `https://stockmarketwords.com/zh-cn/` loads in Chinese
- `https://stockmarketwords.com/raw-ftp-data/` shows today's date in the data table
