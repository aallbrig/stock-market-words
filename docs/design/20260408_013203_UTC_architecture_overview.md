# Architecture Overview

**Status:** Design / Foundation
**Last updated:** 2026-04-08

This is the one-page system map for stockmarketwords.com. Read this first.
Other foundation docs go deeper into individual subsystems.

---

## What this project is

stockmarketwords.com is a free tool that lets a user paste any text (a news
article, an earnings transcript, a Reddit thread) and instantly extract every
stock ticker symbol hidden inside it, then enrich those tickers with strategy
scores and key metrics. The site also publishes data tables, educational
articles, and a financial glossary. The audience skews toward Singapore.

## The three subsystems

```
+----------------------+      +----------------------+      +----------------------+
|  Python CLI          | -->  |  SQLite database     | -->  |  Hugo static site    |
|  (python3/)          |      |  (data/market_data.db) |    |  (hugo/site/)        |
|                      |      |                      |      |                      |
|  ticker-cli ...      |      |  6 tables, ~8k       |      |  English + zh-CN     |
|  Click commands      |      |  tickers, daily      |      |  Multilingual via    |
|  Yahoo Finance API   |      |  metrics, 5 strat-   |      |  Hugo's built-in     |
|  NASDAQ FTP          |      |  egy scores          |      |  i18n                |
+----------------------+      +----------------------+      +----------------------+
         |                                                           ^
         |                                                           |
         |     +--------------------------------------------+        |
         +-->  |  JSON build artifacts                      |  ----- +
               |  hugo/site/static/data/*.json              |
               |  hugo/site/data/*.json                     |
               |  (all_tickers, strategy_*, raw_nasdaq, ..) |
               +--------------------------------------------+
```

1. **Python CLI** (`python3/`) — pulls raw data from the NASDAQ FTP and Yahoo
   Finance, scores tickers against five strategies, writes the JSON files Hugo
   consumes. Click-based, entry point `ticker-cli`. Run after every trading
   day. See [`data_pipeline.md`](./20260408_013203_UTC_data_pipeline.md).

2. **SQLite database** (`data/market_data.db`) — the durable store. Six tables
   covering ticker identity, daily metrics, strategy scores, and pipeline
   run/step history. Schema is in `python3/schema.sql`. See
   [`database_schema.md`](./20260408_013203_UTC_database_schema.md).

3. **Hugo static site** (`hugo/site/`) — the public website. Multilingual
   (English + Simplified Chinese), uses Hugo's built-in i18n via
   `hugo/site/i18n/{en,zh-cn}.toml` and `.zh-cn.md` content variants. Ticker
   detail pages are generated by a content adapter (`_content.gotmpl`) that
   reads the JSON the Python CLI produces. See
   [`i18n_architecture.md`](./20260408_013203_UTC_i18n_architecture.md).

## What's hand-authored vs CLI-generated

| Content | Source | Notes |
|---|---|---|
| Articles (`hugo/site/content/articles/*.md`) | Hand-authored | Editorial. Not touched by the CLI. |
| Glossary (`hugo/site/content/glossary/*.md`) | Hand-authored | Editorial. Not touched by the CLI. |
| Strategy detail pages (`hugo/site/content/strategy-*.md`) | Hand-authored | Intentionally manual to preserve educational quality. See comment in `python3/src/stock_ticker/hugo_generators.py:429-446`. |
| About / methodology / privacy / contact | Hand-authored | Standard prose pages. |
| `raw-ftp-data.md`, `filtered-data.md` | CLI-generated | Written by `ticker-cli hugo pages`. |
| Ticker detail pages (`/tickers/<symbol>/`) | CLI data + Hugo content adapter | Data file `all_tickers.json` is written by `ticker-cli hugo all-tickers`; pages are materialized at Hugo build time by `hugo/site/content/tickers/_content.gotmpl`. |
| Strategy JSON files (`strategy_*.json`) | CLI-generated | Powered by `ticker-cli hugo strategies`. Consumed by `portfolio-extractor.js` in the browser. |

## The trading-day loop

After US markets close, run `ticker-cli run-all` (or its constituent steps:
`sync-ftp` → `extract-prices` → `extract-metadata` → `build` → `hugo all`).
This refreshes the database, regenerates the JSON build artifacts, and the
Hugo build picks them up on the next deploy. See
[`data_pipeline.md`](./20260408_013203_UTC_data_pipeline.md) for the full
command list and what each writes.

## The home-page tool

`/` and `/zh-cn/` render `hugo/site/layouts/index.html`, which loads
`hugo/site/static/js/portfolio-extractor.js` (~857 lines). User pastes text,
clicks **Extract Tickers**, and the JS runs ticker extraction (with a Web
Worker at `portfolio-worker.js`) against the strategy JSON files in
`hugo/site/static/data/`. All UI strings come from
`hugo/site/layouts/partials/i18n-js.html`, which injects a `window.I18N`
object built from the `hugo/site/i18n/{lang}.toml` files.

## Where things live (top-level cheat sheet)

```
hugo/site/                      Hugo static site
  hugo.toml                       Site config (incl. [languages])
  content/                        Markdown content (en + .zh-cn.md variants)
  layouts/                        Hugo templates
  static/js/                      Browser JS (portfolio-extractor.js, TickerEngine.js, ...)
  static/data/                    JSON files consumed by browser JS
  data/                           JSON files consumed by Hugo at build time
  i18n/                           en.toml, zh-cn.toml — translation strings
python3/                        Python CLI (ticker-cli)
  src/stock_ticker/cli.py         Click commands (entry point)
  src/stock_ticker/database.py    SQLite operations
  src/stock_ticker/hugo_generators.py  Writes JSON + .md for Hugo
  src/stock_ticker/extractors.py  Yahoo Finance fetchers
  src/stock_ticker/ftp_sync.py    NASDAQ FTP sync
  src/stock_ticker/builders.py    Strategy scoring
  schema.sql                      SQLite schema
data/market_data.db             SQLite database (production data)
docs/design/                    Foundation docs (you are here)
docs/specs/                     Per-feature specs
docs/research/                  Exploratory research notes
tests/                          JS test suites
.github/                        GitHub Actions + copilot-instructions.md
```
